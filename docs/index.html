Atlas es una primitiva de estado compartido persistente en Solana. Un World es cualquier espacio donde múltiples actores interactúan con reglas verificables on-chain — un juego, una DAO, un marketplace, un protocolo DeFi. Instálalo, crea un mundo, y en minutos tienes estado compartido que nadie puede alterar retroactivamente.

La primitiva

No es un motor de juego. Es la memoria del mundo.

Cada World tiene estado, actores y reglas — verificables on-chain, sin servidor central. El historial de cada epoch queda congelado para siempre; el presente sigue evolucionando.

Es.

Estado

Lo que existe en el mundo y puede cambiar — recursos, propuestas, listings, posiciones.

Ac.

Actores

Players con identidad on-chain vía PDA, historial y puntuación verificable por epoch.

Re.

Reglas

Resource types, cooldowns y límites configurados por el developer, aplicados por el contrato.

Ep.

Epochs

Ciclos de tiempo donde el mundo se agota, se resetea, y el leaderboard anterior persiste.

Cómo funciona

Cuatro instrucciones, un ciclo completo.

Todo mundo pasa por el mismo flujo, sin importar si es un juego o una DAO.

01

createWorld()

Define tus resource types, visibilidad y duración de epoch. El contrato asigna un world_id único.

→
02

mint.player()

Cada wallet crea su Player vía PDA — identidad y progreso verificables on-chain.

→
03

resource.collect()

Cooldown y rate limiting verificados on-chain. El leaderboard se reordena sin sort completo.

→
04

WorldReset

Al agotarse o expirar, el epoch avanza. El leaderboard anterior queda congelado como historial.

→
05

closeWorld()

Cuando el mundo ya no se usa, ciérralo y recupera el rent. El ciclo de vida completo, sin cuentas huérfanas.

Tres líneas

La misma API, cualquier narrativa.

Lo único que cambia entre un juego y una DAO es qué significan tus recursos.

const { worldId } = await atlas.world.create({ name: 'Mi Juego', worldType: WorldType.Gaming, visibility: WorldVisibility.Public, totalResources: 500, epochDuration: '7d', resourceTypes: [ { id: 0, name: 'wood', points: 1, cooldownSeconds: 5 }, { id: 1, name: 'stone', points: 3, cooldownSeconds: 10 }, { id: 2, name: 'gold', points: 5, cooldownSeconds: 30 }, ] }) // atlas.player.mint({ worldId, name: 'Hero' }) // atlas.resource.collect({ worldId, resourceTypeId: 0 })
const { worldId } = await atlas.world.create({ name: 'Mi DAO', worldType: WorldType.Dao, visibility: WorldVisibility.Public, totalResources: 100, epochDuration: '30d', resourceTypes: [ { id: 0, name: 'vote', points: 1, cooldownSeconds: 86400 }, { id: 1, name: 'proposal', points: 10, cooldownSeconds: 604800 }, ] }) // Cada 'recolecta' de tipo 'vote' es un voto verificable on-chain.
const { worldId } = await atlas.world.create({ name: 'Mi Marketplace', worldType: WorldType.Marketplace, visibility: WorldVisibility.Public, totalResources: 1000, epochDuration: '1d', resourceTypes: [ { id: 0, name: 'listing', points: 1, cooldownSeconds: 3600 }, { id: 1, name: 'featured', points: 5, cooldownSeconds: 86400 }, ] }) // El leaderboard diario se convierte en tu ranking de sellers destacados.

Honestidad técnica

Qué está listo y qué sigue pendiente.

Preferimos que lo sepas ahora que que lo descubras en producción.

RESUELTO Emergency stop. pauseProtocol() bloquea escrituras en todo el protocolo sin afectar lecturas.
RESUELTO Instrucciones de desarrollo bloqueadas por default. close_protocol y admin_set_world_count requieren un feature flag de compilación explícito — estructuralmente imposible enviarlas a mainnet por descuido.
RESUELTO Migración de autoridad sin redeploy. transferProtocolAuthority() y transferWorldAuthority() habilitan mover el control a un multi-sig (Squads) en cualquier momento.
RESUELTO Cierre de mundos. closeWorld() devuelve el rent cuando un mundo ya no se usa.
PENDIENTE Auditoría externa. El contrato no ha sido auditado todavía. Trátalo como beta en mainnet hasta que exista un reporte público.
PENDIENTE Multi-sig real. La instrucción existe; falta configurar el Squads Vault y ejecutar la migración.

Ciclo de vida completo

Crea, prueba y cierra — sin cuentas huérfanas.

La mayoría de los proyectos on-chain se olvidan del "cerrar". Atlas no.

01

Estimado antes de firmar

create-world calcula el rent real en bytes antes de pedirte confirmar — nunca firmas a ciegas.

02

Keeper de desarrollo

atlas-cli watch avanza epochs automáticamente mientras pruebas — no pienses en infraestructura todavía.

03

Errores que explican

Nada de códigos crípticos — "el mundo está agotado, corre advance-epoch" en vez de EpochMismatch (6006).

04

Cierre con reembolso

atlas-cli close-world devuelve el rent cuando terminaste de probar. Nada queda pagando renta para siempre.
